The Vaad Management App is a Streamlit web application for handling apartment building finances and residents.
system was built by Eliram Lugassi
you can contact him in vaadmanagmnet@gmail.com

Entities:
- **Buildings** contain many apartments.
- **Apartments** belong to a building and can have multiple residents.
- **Residents** hold contact info, role (owner or renter) and active status.
- **Expected charges** store monthly fees to be collected from each apartment.
- **Transactions** record actual payments by residents.
- **Suppliers** and **Expenses** track costs for a building. Payments link expenses to specific months.
- **Users** log in with a role of `admin` or `user`.

Roles and permissions:
- **Admin** users can manage all data including buildings, apartments, residents, fees, expenses and other users.
- **User** accounts can view data only for buildings assigned to them and cannot access the admin panel.

Structure:
- The app uses modular pages: `dashboard`, `buildings`, `invoices`, `suppliers`, `expenses`, `transactions`, `my_profile`, and an `admin_panel` for administrators.
- `st.session_state` stores the current username, role, language, and other selections such as `building_id`.
- Friendly names for these pages are defined in `localization.py` so the UI can display them in different languages. Examples of the English/Hebrew mappings are:
  * `dashboard` → "Dashboard" / "לוח בקרה"
  * `buildings` → "Buildings" / "בניינים"
  * `invoices` → "Invoices" / "חשבוניות"
  * `suppliers` → "Suppliers" / "ספקים"
  * `expenses` → "Expenses" / "הוצאות"
  * `transactions` → "Tenets payments" / "תשלומי דיירים"
  * `my_profile` → "My Profile" / "הפרופיל שלי"
  * `admin_panel` → "Admin Panel" / "ממשק ניהול"


work process:
the users in the app are the HOA representative in their building, managing tenants payments, expenses, reports, suppliers and more
user should update its details in "my profile" page
user should create a building and then create apartments
after creating apartments, the user should create residents and the set who is the active resident,
active residents will get e-mails and payments will be under their name.
after creating residents,
a user must set the monthly to the apartments either in bulk or individually in the "buildings" page.
before adding expenses, a user must enter its suppliers in suppliers page
if not entered, the expenses won't be recorded.
you can add expenses in the expenses page and tenets payments in the transactions (tenets payments) page.

# VAAD MANAGEMENT APP - USER NAVIGATION GUIDE

This guide helps users navigate and understand the core features of the Vaad Management App.

---

1. LOGIN AND SESSION HANDLING

---

* Enter your username and password to log in.
* Sessions will automatically expire after 3 minutes of inactivity.
* If your session expires, simply log in again.

---

2. LANGUAGE SELECTION

---

* In the sidebar, you can select your preferred language:

  * English
  * Hebrew

---

3. MAIN MENU NAVIGATION

---

The sidebar provides access to the following pages:

* Dashboard
* Buildings
* Invoices
* Suppliers
* Expenses
* Transactions
* My Profile

Admins also have access to:

* Admin Panel
* View as User / View as Admin toggle

---

4. DASHBOARD

---

* Shows financial overview:

  * Paid vs Expected Amounts
  * Total Paid Expenses
  * Pending Expenses
  * Net Balance
* Cash Flow Chart (last 6 months)
* Manual Bank Reconciliation Tool
* Tables for:

  * Unpaid Expenses
  * Unpaid Apartments
  * Expense Breakdown

---

5. BUILDINGS PAGE

---

* View list of buildings assigned to you
* Add or delete buildings (admin only)
* Add or update apartments
* Add, edit, delete residents
* Set active residents (only one per apartment)
* Update monthly fees:

  * In bulk for all apartments
  * Individually per apartment
* Send messages to all residents by email

---

6. TRANSACTIONS PAGE

---

* View and manage all payment records
* Filter by building, apartment, year, and month
* Add individual payments
* Edit or delete existing payments
* Bulk insert payments for multiple apartments
* View total amount paid

---

7. INVOICES PAGE

---

* Select a building and a month to view transactions
* Send invoices in bulk or individually via email
* Invoices are generated as PDF files and include:

  * Resident name and apartment number
  * Payment method and amount
  * Building contact info (from My Profile)
* Email is sent using the contact provided in each transaction

---

8. SUPPLIERS PAGE

---

* View all suppliers
* Add a new supplier (name, expense type, segment)
* Edit or delete supplier records

---

9. EXPENSES PAGE

---

* Filter by:

  * Building
  * Expense Type
  * Status (pending/paid)
  * Year and Month
  * Receipt ID
* Add new expenses
* Edit or delete existing records
* Optional: Upload scanned receipt for data extraction (if enabled)
* View total expense summary

---

10. MY PROFILE PAGE

---

* Update your own email and password
* Update phone and email contact info for buildings you manage
* Preview how your contact info appears in PDF invoices

---

11. ADMIN PANEL (Admin Only)

---

* View all registered users
* Track users active in the last 5 minutes
* Edit user email, role, and password
* Delete users (cannot delete yourself or other admins)
* Assign buildings to users
* Create new user accounts
* Start onboarding wizard for new users

---

12. ONBOARDING WIZARD

---

Guided first-time setup flow for new users:

1. Update your contact info
2. Add a building
3. Add apartments
4. Add residents
5. Set active residents
6. Set monthly fees
7. Add initial transactions
8. Add expenses

You can skip and return to any step.

---

13. TROUBLESHOOTING

---

Problem: "No active resident found"
Cause: Apartment has no active resident
Fix: Go to Buildings page and set an active resident

Problem: No invoices show
Cause: No paid transactions for selected month
Fix: Confirm transactions are recorded for that period

Problem: Session expired
Cause: Inactivity for more than 3 minutes
Fix: Log in again

Problem: No data showing
Cause: Incorrect filters applied
Fix: Reset filters and try again

Problem: Receipt analysis missing
Cause: Feature disabled in some versions
Fix: Use manual data entry in the Expenses page

---

14. FREQUENTLY ASKED QUESTIONS (FAQ)

---

Q: I forgot my password. How can I reset it?
A: Contact your admin. Admins can reset your password in the Admin Panel.

Q: Can I access more than one building?
A: Yes. Your admin can assign multiple buildings to your user account.

Q: Why do invoices include the wrong contact details?
A: Go to "My Profile" and update the contact phone/email for each building.

Q: How do I change my language preference?
A: Use the language dropdown at the top of the sidebar.

Q: Can I send invoices only to specific apartments?
A: Yes. In the Invoices page, use the checkboxes to select specific rows before clicking "Send".

Q: What file types are supported for receipt uploads?
A: JPEG, PNG, PDF (if receipt scanning is enabled in your system).

Q: What happens if I delete a building?
A: All apartments, residents, transactions, and expenses associated with that building are deleted.

Q: Who can see my data?
A: Only admins and users assigned to your building can see your building's data.

---

Need further help?
Contact your system administrator or support team.

• תשלומי ועד על דירות קבלן שלא נמסרו – הקבלן חייב בתשלום השוטף, אך בחוזים רבים מופיעה החרגה להוצאות חריגות. יש לבדוק את הסכם הרכישה; עו״ד הגר חדד.
• ביטחון: אירוע חדירה בבניין (רח׳ יצחק רבין) – רעולי‑פנים תועדו; חשיבות סגירת דלתות ושיתוף צילומי מצלמות.
• ביטוח לעמדות טעינה לרכב חשמלי – חובה להוסיף כיסוי צד ג׳ בפוליסת דירה או לרכוש פוליסה ייעודית (‎200‑400 ₪ בשנה).
• שיטות חיוב חשמל לעמדות טעינה:
  – מומלץ משיכת קו מהדירה או ממחסן פרטי (גם אם דורש תוואי ארוך/תלת‑פאזי).
  – פתרון ביניים: מערכת “עמדה חכמה” עם ניהול צריכה וגבייה אוטומטית דרך מונה הוועד.
  – הוועד יכול להתנות התקנה באישור ≥50 % מהדיירים; מנגנון קיזוז/גבייה מוסדר מראש; לדרוש אחריות על נזקים ועמידה בתקני חשמלאי בודק.
  – שימוש זמני בחשמל משותף והחזר חודשי נחשב “שירות” ועלול לעורר סוגיות מע״מ/מס.
• ספקי גז – מעבר מאמישראגז לגז‑גל הוריד מחיר מ‑‎45‑60 ₪/מ״ק ל‑‎15 ₪ ומחיר אחזקה מ‑‎30‑50 ₪ ל‑‎12 ₪; אפשר להפעיל לחץ על הספק הקיים או להתאגד לשינוי ספק.
• חשמל – יוזמה להתאגדות ועדי‑בתים לשיפור תנאי החשמל מול ספקים פרטיים; דרושים נתונים על מספר דירות.
• בקשות שירות נפוצות:
  – חברת ניקיון לחניון תת‑קרקעי.
  – תליית מראה פנורמית Ø 80 ס״מ בכניסת חניה.
  – חידוש פנים מעלית (הסרת שטיח קיר והחלפת גופי תאורה ל‑LED).
  – החלפת קו מים ראשי רקוב; המלצה לאינסטלטורים (דרך “המקצוענים” או קבלן פרטי).
  – תכנון מחודש לכניסת בניין מאלומיניום כולל דלת, תיבות דואר, אינטרקום מצלמה.


=== עדכון מאי‑יוני 2025 ===
* **אינסטלציה / ביוב** – מומלץ להזמין אינסטלטור עם ביובית לטיפול בסתימות או בצינור שבור בבניין. במקרה קיצון יידרש חפירה והחלפת קו (7‑8.5.25).
* **איטום גגות** – לוודא שקבלן האיטום אמין, נותן אחריות כתובה ומגיע לתיקונים. אחריות של 10 שנים לא שווה ללא זמינות (8‑9.5.25).
* **תקנון הבית המשותף**
  - ניתן להזמין תיק בית משותף באתר משרד המשפטים (27 ₪).
  - שינויים כמו גבייה שווה או הצמדת שטח משותף מצריכים רוב מיוחד (לעתים 100 %).
* **מחסנים ורכוש משותף** – שימוש בלעדי על‑ידי דייר אסור ללא הצמדה רשומה. לפני פינוי עצמי: מכתב התראה → מפקחת (5.6.25).
* **עמדות טעינה לרכב חשמלי**
  - עדיף חיבור ישיר משעון הדירה או מערכת חכמה עם חיוב אוטומטי; לא למשוך חשמל מהבניין ולגבות אחר כך.
  - לעדכן ביטוח דירה בכיסוי צד ג׳ לעמדה; ללא ביטוח – כ‑200‑400 ₪ לשנה.
  - הרחבת תשתית (תלת‑פאזי) דורשת החלטת אסיפה וחלוקה הוגנת (7.5.25).
* **הדברת קיץ** – ריסוס בורות ביוב של הבניין ודירות גן; לפתוח קריאה במוקד העירוני לריסוס ציבורי. לבדוק בורות בבתי גן לפני כיסוי דשא סינתטי (10.6.25).
* **חירום וממ״ד**
  - ציוד: נעליים, תיק חירום (בגדים, תרופות, דרכונים, כסף), פטיש ≥ 2 ק״ג, לום, כפפות עבודה, מסכות.
  - להשאיר פתק על הדלת אם יצאתם; ועד שומר רשימת דיירים ומספרי טלפון (16‑17.6.25).
* **מצלמות אבטחה** – חייבות אישור אסיפה; לצלם רק שטח משותף; הגישה לקבצים בידי הוועד אלא אם הוחלט אחרת (19.6.25).
  - תקרית עובדי ניקיון שהסתובבו בחניון מדגישה את הצורך במצלמות.
* **תחזוקת מעליות** – המלצה על סיגמא מעליות (שמעון פרץ ✆ 050‑725‑2707); מחיר כ‑30 % נמוך מהצעות מתחרות (5.6.25).
* **חשבון בנק לוועד** – בלאומי נדרשות עמלות; רצוי לבצע השוואת עלויות לפני פתיחה (25.5.25).
